A pip down memory $PATH

Pip is the package manager for Python. To install a package via pip

pip install <packagename>

However, the above command only installs the package for that user for that session of the shell. This I realized when I pip installed a package but when I tried to import it in my python program, got an error that the package does not exist. I was mighty confused because I definitely installed the package a day ago!

This led me to troubleshoot and learn about an environment variable called PATH. Environment variables are global variables that contain information about the login session, stored for the system shell to use when executing commands. A  really good article that helped me understand more about this is

https://astrobiomike.github.io/bash/modifying_your_path#the-path-demystified

You can see all your environment variables by running the command env. There you will see not only PATH but also other env variables like username, lang, shell, etc. Mine looks like this

/.local/bin:/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin:/usr/games:/usr/local/games:/snap/bin

So what is PATH? PATH is all the locations the computer checks when running commands and scripts (ones that start with the #!/bin/bash shebang). Each location is distinguished by a colon (“ : ”). A neat command to parse through the output and display each location in a new line is this

echo $PATH | tr “:” “\n” 

This will take the output of echo $PATH and pipe that into the tr command which substitutes every “:” for a \n or new line

https://www.geeksforgeeks.org/tr-command-in-unix-linux-with-examples/

/.local/bin

/usr/local/sbin

/usr/local/bin

/usr/sbin

/usr/bin

/sbin

/bin

/usr/games

/usr/local/games

/snap/bin

So when I was pip installing a package, it was getting saved in the .local/bin PATH which is only available to the current shell session. Thats why I could not find my packages the next day. To install globally, I needed to do

sudo pip install <packagename>

Also when I did which pip the location turned up as .local. So I uninstalled pip by the following command

pip uninstall --user pip


Now, I also don’t want to install any pip packages locally in the future. So either I use sudo every time OR an easy, but probably not an elegant, fix is to create an alias for pip in my ~/.zshrc file

alias pip=’sudo pip’
