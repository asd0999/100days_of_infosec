Bash scripting (part 1)

Took more than a few days off from the project due to COVID-19 mayhem. Now that life is a little more settled (not really) I thought I’d get back into it with some bash scripting.

Bash scripting (actually zsh is the shell I use) is a really powerful tool and I’ve seen folks do some crazy automation using it. It’s been on my list of things to learn for a while so let’s get into it!

Fun fact I learnt while going over some of the shell basics, bash actually stands for Bourne Again Shell. That’s pretty cool if you ask me!

So I had a script from a bash scripting workshop I attended some time ago. Here’s the script:

#!/bin/bash

alias gtw='cd $working_dir'

wd(){

            working_dir=$(pwd)

            echo "You're working dir is $working_dir"

}

The script makes it easier to jump between directories, it’s like saving a bookmark for a directory, like a speed dial. It has a function wd() which saves the current working directory $(pwd) in a variable and then aliases the command gtw to cd into the directory

However, the script didn’t work as shown to me during the workshop and that’s because I missed a crucial step. Even though this script exists and is part of my $PATH, the wd function is not callable unless the script iis running and even if I run the script manually, it only runs momentarily. So I had to load the function into my current shell session somehow so that it can be called whenever during that session.

The source command can be used to load any functions file into the current shell script or a command prompt.

Doing this would make the wd function available to be called whenever irrespective of whether the script itself is running or not.

So in my .zshrc file, I appended:

source working_dir.sh

working_dir.sh is the name of the script. So now every time a new zsh session is launched, the functions contained in the working_dir.sh script are loaded into the session. And the script works just fine!
