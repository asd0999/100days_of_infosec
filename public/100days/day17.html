Docker

What is docker? It’s a container. I've heard that answer so many times and my next question is always "what's a container?" lol. Today I tried to understand what docker is and why it may be a useful tool.  

VM vs Docker
I’ve known about VMs for a while, I’ve used one (VirtualBox) to run Kali or Windows on my Mac. VMs virtualize the hardware and run an entire OS on the virtualized hardware. There can several VMs running on a machine and that’s how one computer can act as several (servers) on services such as AWS / Digital Ocean.

Docker containers, on the other hand, virtualize OS. They create a virtual file system for the application / service to run off of the host OS. Multiple containers can run on the same OS without replicating the OS code which is why they are so lightweight. Containers can be isolated from other containers as well as the host machine depending on their configuration.

The different pieces involved in running docker containers are :

>>> Registry - Repository of images that you can pull from, check out hub.docker.com


>>> Image - A template of instructions for creating a docker container. Custom images can be built using Dockerfiles. Each instruction creates a new layer of the image


>>> Container - An instance of the image that can be run on the host OS
