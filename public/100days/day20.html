Format string vulnerability

In C, a printf statement looks like this :

printf (“Hi %s, you id number is %d.\n”, Jay, 5);

The above would print :

Hi Jay, you id number is 5

So what happens when you give the following command?

printf(%p) //$p is a pointer

Wargames —  leakme

Applying the above information, I solved a format string vuln challenge. This is the program :

The format string vulnerability

Notice the 0x70257025’s, those are the %p’s being stored on the stack. Neat!

>>> Binary Ninja

When I open the executable with binary ninja, I can see that there is a comparison happening in the Challenge ( ) function which decides which instruction to jmp to next

The Comparison is happening between the values stored in address [rbp + 0x54] and [rbp + 0x64]

>>> gdb

Disassemble challenge ( ) and set breakpoint at the cmp instruction

disass challenge

b *  0x0000000000400942

run

>>> When prompted I give the following input

%p%p%p%p%p%p%p%p%p%p%p%p%p%p%p%p%p%p

>>> Since I know that the values that will be compared, let’s look at those at this breakpoint

x $rbp-0x54

x $rbp-0x64

Ah! So my input is in [rbp - 0x54] and [rbp - 0x64] must be the random integer! Let’s find that in the output of the %p’s above

Now that I know where in the memory the random integer is being stored, I can leak it again and this time I can convert it to decimal and input it as the magic number! So let’s do the whole thing again

run

%p%p%p%p%p%p%p%p%p%p%p%p%p%p%p%p%p%p

Find the value and convert to decimal and enter it at the prompt

let’s look at the two values being compared again

x $rbp-0x54

x $rbp-0x64

It’s the same! That means we shall now jmp to the win instruction!
