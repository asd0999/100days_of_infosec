CTFd —  linking an external libc library

So for a CTF challenge I needed to link an external libc library to run the challenge locally. This was because the challenge needed to build a ROP chain using libc functions

What is libc? Libc is the standard C library from where programs borrow standard functions like gets and puts

I tried adding LD_PRELOAD in my pwntools script but that kept crashing

Then I found this link

https://www.reddit.com/r/securityCTF/comments/70svdk/pwn_linking_the_challengeprovided_libcso_instead/

And I realized that the environment are set up differently systems (OS) so I tried LD_LIBRARY_PATH instead and that worked.

p = process('./rop', env = {'LD_LIBRARY_PATH' : './libc-2.23.so'})

This process also led me on to discovering where the libc libraries are stored on my computer locally and how many there are!

For me the path is: /usr/lib/x86_64-linux-gnu/ and there are over a 100 libc libraries there! (I use ubuntu 18.04)
