Shell scripting 101

As a continuation of yesterday’s PATH exploration, I decided to experiment with some shell scripting or bash scripting

A bash script starts with the following (aka the shebang) :

#!bin/bash

Then come the commands the script should execute. I’m creating a simple script to make git commits and push them to my 100days repository. This is how that is done

git add .
read -p “Commit message: “ message

git commit -m “$message”
git push origin master

The above will prompt me for an input (because of the -p flag) and saves that input in the variable message which is later referenced as $message

Now this script is in my git repo folder but that directory is not in my PATH so when i try to execute the script, my computer will not look in that directory, and hence, won’t find my script to execute. Here’s what to do to add a PATH to the $PATH environment variable

export PATH=”$PATH:<full_path_to_the_directory>”

Ex:
PATH="$PATH:/home/tushar/girgit/100days_of_infosec/16_day"
export PATH

This will append the directory at the end of the existing PATH. However, this is only valid for the current session

To export this location permanently, edit the ~/.bash_profile or in my case the ~/.zshrc file  

Copy pasting the export command at the end of the file creates a permanent addition to the PATH with my 100days directory and I can now run the script from wherever on my computer

Note: Even though the script can be executed from any directory, git will throw an error if I’m not in the git repository
