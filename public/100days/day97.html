Bash scripting (part 4)

Variables

FRUIT=apple

FRUIT[1]=orange

FRUIT is assumed to be FRUIT[0]

If I assign FRUIT[0]=grape, the value of FRUIT is no longer apple

readonly FRUIT ⇒ this means FRUIT or FRUIT[0] is now a readonly variable and its value cannot be changed

Arrays can be assigned manually or like this:

FRUIT=(apple orange “passion fruit” peach)

echo $FRUIT ⇒ apple

echo ${FRUIT[2]} ⇒ passion fruit

echo ${FRUIT[*]} ⇒ apple orange passion fruit peach

Local variables (like the ones above) can be made into environment variables by “exporting” them

Example:

NAME=kalki

export NAME

env | grep NAME ⇒ kalki
