Setting up Ubuntu 18.04 on a Windows machine

I recently got a Windows machine so I could dual boot it with Linux. Here is the process I followed to set up the dual boot as well as installing essential tools using the apt packet manager

Part II - apt install -y <essential-tools>

<<firstly, stop the unnecessary daemons/processes>>

systemctl status snapd

systemctl disable snapd

systemctl stop snapd

systemctl status snapd

systemctl disable apparmor

systemctl stop apparmor

systemctl status apparmor

<<this is where the system daemons can be found>>
cd /etc/systemd/system

<<install tree>>

apt update

apt install -y tree

<<my auto update was on and im guessing thats what was causing the 'lock' issue, just rm the file>>

rm /var/lib/dpkg/lock-frontend

apt install -y tree

rm /var/lib/dpkg/lock

apt install -y tree

tree

<<install zsh and git>>

apt install -y zsh git

git clone https://github.com/ohmyzsh/ohmyzsh.git ~/.oh-my-zsh

cp ~/.oh-my-zsh/templates/zshrc.zsh-template ~/.zshrc

nano .zshrc

chsh -s $(which zsh) <<change default shell>>

<<zsh plugins>>

cd .oh-my-zsh/custom/plugins

git clone https://github.com/zsh-users/zsh-syntax-highlighting

nano ~/.zshrc

plugins=(

  git

  zsh-syntax-highlighting

  <<add whatever other plugins here later>>

)

<<cool script to try different themes>>

cd .oh-my-zsh/tools/

./theme-chooser.sh

<<installing essential tools like curl>>

apt install build-essential

apt install nasm cmake

<<vscode install>>

download .deb file from the website

dpkg -i ~/Downloads/code_1.41.1-1576681836_amd64.deb

<<docker>>

curl https://get.docker.com/ | sh

sudo usermod -aG docker <username>

<restart computer>

cd ~/opt

sudo git clone https://gitlab.com/b1g_J/dockerfiles.git

sudo chown <username>:<username> -R dockerfiles

sudo apt install -y docker-compose

<<giving permission to any app to open a GUI screen>>

nano ~/.zshrc

xhost + &> /dev/null

Note:
/dev/null is the void, the black hole, what goes in doesn’t come back out
