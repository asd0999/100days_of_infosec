CTFd —  Git Got Good (part 1 - GOT)

Going to work with GOT today! GOt stands for Global Offset Table.

A program may contain several standard functions like puts, fgets, etc. These are taken from standard libc libraries. The libraries can be included into the binary when compiling the program. Thisi known as static linking and makes the binaries huge! Or the program can leverage the libraries that already exist on the system. This is called dynamic linking. Since, the standard functions are not part of the program itself, it needs to find the relevant functions in the libc library before it executes it. This is what GOT is used for. GOT is like a trampoline which sends the program on to the right addresses for a libc function.

GOT is a table which contains addresses and function names (or symbols). Essentially when the program goes looking for puts, GOT tells to go to a certain address to find it

Useful link:  https://systemoverlord.com/2017/03/19/got-and-plt-for-pwning.html

So I decided to run the binary with gdb and see if i can follow the program to the resolution of the printf function in libc

Note: set follow-fork-mode parent is important, if set to child the program will crash because the bin/dash function will make it follow the child process and die when that process is done (time is returned) 

First I disassembled main to see the address where printf@plt is being called which would be my break point. printf@plt means the program will first refer to the PLT or the Procedure Linking Table which will point is to the GOT where it will eventually find the printf function address in libc

Continue with ni (next instruction) or si (step into) when there is a call, gdb will take you through a lot of stuff which I don't fully understand but you will eventually land up at the printf function in libc.

In this case the address of the function is : 0x7ffff7a48e80

info symbol <address / pointer>
