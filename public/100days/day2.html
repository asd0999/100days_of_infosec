picoCTF 2019 — Reverse Engineering (continued)

vault-door-3

So first of all, vault-door-2 doesn’t exist. I downloaded VaultDoor3.java. To run a java program, I used javac (OSX ships with it)

$ javac VaultDoor3.java

This creates a .class file which is an executable. To run:

$ java VaultDoor3

This ran the program which prompted me for a password. Now the reversing part. The checkPassword( ) function takes user input as the argument ‘password’ and creates a new char[ ] array ‘buffer’. The characters of ‘buffer’ are populated from the characters of ‘password’ but in a different order through 4 FOR loops after which it compares the ‘buffer’ with a predetermined string. So essentially, I gotta reverse engineer the the FOR loops so to enter a password which when jumbled up gives me the predetermined string to result in a return True;  

FOR loop 1 - same as the first 8 characters

FOR loop 2 - second 8 characters but in reverse order

FOR loop 3 - 8 characters starting at character 30 in reverse order, skip one

FOR loop 4 - 8 characters starting at character 17 skip one


This one, even though not really hard, threw me off a little ‘cause I kept making silly counting mistakes. I had to make notes finally to solve it correctly

Ran the program and entered the flag to get an “access granted”

Flag: picoCTF{jU5t_a_s1mpl3_an4gr4m_4_u_aa931e}
