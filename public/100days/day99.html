DNS zone transfer

Zone transfer is copying the DNS information from a primary DNS server to a secondary DNS server

It is meant for smooth functioning of the DNS servers but sometimes DNS servers can be misconfigured and an attacker can dump the valuable DNS information

DNS zone transfer can be done using host, dig or nslookup commands

Host

host -t ns zonetransfer.me  → to find out name server

→ nsztm1.digi.ninja and nsztm2.digi.ninja

host -l zonetransfer.me nsztm1.digi.ninja

→ and you get all the DNS zone information

Dig
dig -t ns zonetransfer.me  → nsztm1.digi.ninja

dig axfr zonetransfer.me @nsztm1.digi.ninja

Nslookup (windows)
C:\> nslookup

> set type=any

> zonetransfer.me → nsztm1.digi.ninja

> server nsztm1.digi.ninja

> ls -d zonetransfer.me

Some countermeasures for DNS enumeration:

  • Disable zone transfer to untrusted hosts
  • Do not publish private hosts/IP addresses in DNS zone files of public DNS server
  • Use standard admin contact info for DNS registration to avoid social engineering attack or use a service/agent to Shield your information
