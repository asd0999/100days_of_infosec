lets analyse...

The read function call

read (1, rbp-0x50,0x40) reads from fd 1 for 0x40 bytes and stores the value in rbp-0x50 address. On success, it returns a 0 in rax

So rbp-0x50 has the value I enter

The next few instructions do the following :

rbp-0x54 = 0

eax = rbp-0x54

cmp eax with 0x2f

jmp if eax <= 0x2f

eax = 0

eax = al = 1 byte from [rax+0x601080]

eax = not eax

edx = eax

eax = 1 byte from [rbp-0x50+rax]

Since rax=0, we need to compare the first byte at 0x601080 (edx) with the first byte of the user input (eax)

Clicking on 0x601080, we see that the first byte is 99 or 0x99

binary(0x99) =  10011001

(not  10011001) = 01100110

01100110 = 0x66

So edx = 0x66

char value of 0x66 (or decimal 102) is 'f'

So for the compare instruction to return true, the first character of my user input needs to be ‘f’

Then the program increments rbp-0x54 to 1 and loops back to address 0x004007b1 where rbp-0x54 is moved into eax. The exit condition for the loop is when count becomes greater than 0x2f (or 47)

Ok, so now we know that with every loop, the program will compare the subsequent byte in memory (rax+0x601080) with the subsequent byte in user input (rbp-0x50+rax). Since we know all the byte in memory, let’s write a python script to do the math for us and give us the flag
