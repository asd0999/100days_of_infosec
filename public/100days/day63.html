I’ve often seen permissions being set in terms of numbers like 777 and wondered what that’s about. So today I thought I’’d read up on it and demystify the mystery.

So there are 3 kinds of file permissions (primarily) :

r - read

w - write

e - executable

And these numerically represented as follows :

4 - read

2 - write

1 - executable

I usually use the following command to make a file executable :

chmod +x <file_name>

 

The same permission could be represented numerically :

chmod 777 <file_name>

7 means 4 + 2 + 1 means the file is readable, writeable and executable. But why 3 numbers? Because the numbers represent the permissions for the user, the group and others, respectively.

So permission 751 would mean the following :

        User          rwx

        Group          r-x

        Others          --x

Well, now I know!
