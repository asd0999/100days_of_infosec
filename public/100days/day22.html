Wargames —  leak global

Another variation of the leakme program. This one accepts 99 characters like the original one but this one declares a global variable outside of all the functions.

>>> Local variables get pushed on the stack but global variables are pushed onto the heap memory.

>>> Note : The Stack grows downward (higher address up top, lower addresses to the bottom) but the Heap grows upwards.

So there must be that address where the stack ends and the heap starts. I tried the %15$p input like the last challenge but that was incorrect.

Then I tried %16$p (the next one) and this time the value looked different, it was 8 bytes long (instead of 6 bytes as returned in the above try). Maybe that’s the first value in the heap?!?

Viola, that was it! Converted the hex into decimal the same way to get the magic number!
