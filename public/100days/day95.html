Symbolic vs hard links in Linux

You can link files in Linux such that a file can “exist in two places”. This can be done in two ways:

  • Symbolic link
  • Hard link

Symbolic link creates a file that *points* to the target file, but isnt the same file itself. It has different inode and different permissions

Hard link, on the other hand, creates a mirror of the target file with the same inode and permissions.

In both, the changes in the target file will be reflected in the linked files - with a symbolically linked file because when you cat the linked file, the file would first resolve in the target file and then the content would the cat’d while with a hard link, the file is a mirror image of the target (literally, because they have the same inode - so the system is referring to the same file in two places)
