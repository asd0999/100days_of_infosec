Another reversing problem today. Program flow in Binja :

The main function makes a call to get_number function which contains a strtol function. man strtol revealed the arguments required for each of the functions, the arguments being rdi, rsi and rdx

Description

The  strtol function converts the initial part of the string in nptr to a long integer value according to the given base. Base is the last argument (rdx / edx) which here is 0xa or 10 (decimal)

Return value

The strtol function returns the result of the conversion (which is stored in rax(/eax) by convention

An important instruction to understand :

mov rdx, qword [rax]

***

There is a very important distinction between rax and [rax]. 

rax refers to the value stored in the register but [rax] is the dereferenced rax which points to the value stored in the address stored in rax (assuming it’s a valid address)

***

Now since, rax holds the user input in decimal (ref: strtol function above), the above instruction means => mov the value at the address entered by the user (ie dereferenced rax) into rdx

The next few instructions do the following :

rax = 0xd000dfaceee

cmp rdx, rax (=0xd000dfaceee)

jmp to success branch if equal

Hence, for the success condition to be true, rdx (ie [rax]) needs to be equal to rax or 0xd000dfaceee. Now, 0xd000dfaceee is part of the cmp instruction which is stored in memory so if we can point to the exact address containing the 0xd000dfaceee value (via the user input), it will be dereferenced to the value itself and moved into rdx which will cause the success condition to become true!

Ok, so what is the address of 0xd000dfaceee???

In Binja, press space to open to linear disassembly view

Then press H to see the instructions in hex editor view

So the instruction starts at the highlighted 48 which is the address 0x004008ce. You can also count from the beginning of the line which starts at 0x004008c0 to 8ce which will land at the highlighted 48

You can see the 0xd000dfaceee value highlighted above which is 2 bytes after 0x004008ce which is 0x004008d0

So that’s the address we need to input (in decimal) to dereference the 0xd000dfaceee value and get the success condition to return the flag!

decimal (0x004008d0) = 4196560
