picoCTF 2019 — Reverse Engineering (assembly)

asm3 - dynamic analysis

I re-did the asm3 challenge but this time with dynamic analysis using command line tool gdb 

Flag : 0xa4e1


I couldn’t run the assembly as-is in gdb because .S / .asm files are assembly files that need to be linked to a program like C. Steps :

>>> assembly as given by the CTF is not in the standard intel format so first clean the file

>>> create an elf (Linux executable) object from the .asm file using a command line tool called nasm 
nasm -o asm3.o -f elf asm3.asm

>>> link that elf object (.o file) to the C program using gcc
gcc -m32 -o a.out asm3.o main.c

[or use the make command line tool to automate the above]

After doing that I ran the compiled C program with gdb and dynamically analysed the return value EAX at every subsequent instruction to get the flag. gdb commands :

gdb a.out
b main
r
si
