pwnable.kr —  fd (1 pt)

Solved a pwnable challenge on file descriptors. Interesting challenge. Summary :

ssh into the box

ssh fd@pwnable.kr -p2222

There’s a flag file but cat doesnt work due to permissions

ls -l

Flag file has permissions

-r--r----- 1 fd_pwn         root   50 Jun 11  2014 flag

There’s another file fd

-r-sr-x--- 1 fd_pwn         fd   7322 Jun 11  2014 fd

File is owned by fd_pwn in the fd user group. That means fd can run the executable and due to the set UID tag fd runs as fd_pwn. So this program can be used to read the flag file

fd.c is the C program for the ./fd compiled

So the fd.c program takes a number as an argument and stores that number less 0x1234 in   int fd

Then, the len variable reads from the file descriptor and string compares the value entered. If string entered is LETMEWIN, it cats the flag file

File descriptors :

0 => stdin

1 => stdout

2 => error

Now 0x1234 = 4660

So let’s pass 4660 as the argument to the fd executable

./fd 4660

LETMEWIN

Boom!
