Worked on a reversing problem today. Opened the binary in binja and this is what it looked like :

The main function makes a call to get_number function at 0x004008c5

The get_number function :

man fgets and man strtol revealed the arguments required for each of the functions, the arguments being rdi, rsi and rdx

Description

The  strtol function converts the initial part of the string in nptr to a long integer value according to the given base. Base is the last argument (rdx / edx) which here is 0xa or 10 (decimal)

Return value

The strtol function returns the result of the conversion (which is stored in rax(/eax) by convention

The next few instructions do the following :

[rbp - 0x8] = rax (my input converted to an integer)

eax = 0xdeadbeef

cmp [rbp - 0x8] and rax

jmp if equal to the next success branch

So essentially I needed to give an input which was 0xdeadbeef in decimal ie 3735928559 to go to next success branch

Doing this process for each branch, I had to give more decimal equivalents of hex values that were being compared to eax before the jump instructions to reach the flag
